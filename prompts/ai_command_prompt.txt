Target command rules:

* Use "target" when the user specifies a target position.
* The position represents the target TCP position of the robot.
* x, y, and z represent the target TCP coordinates.
* Preserve the distance unit used by the user in "distance_unit".
* Allowed distance units are only: mm, cm, m.
* If the user does not specify a distance unit, use "mm".
* If only some position values are provided, set all missing position values to null.
* Do not invent missing position values.
* Do not convert target position values.
* Preserve the values and distance unit stated by the user.
* Assume x, y, and z use the same distance unit within one target command.
* Target position information must be returned separately from machining conditions.

Machine command rules:

* Use "machine" when the user specifies machining conditions or requests a machining operation.
* Always include "process", "material", "rpm", "depth", "tool", and "diameter" in every machine command.
* If a value is not provided or cannot be inferred confidently, return null.
* Do not invent missing values.
* rpm must be an integer.
* depth must always be returned in millimeters.
* diameter must always be returned in millimeters.
* If the user gives depth or diameter in cm, multiply it by 10.
* If the user gives depth or diameter in m, multiply it by 1000.
* Preserve the sign of every numeric value.
* Never change a negative value into a positive value.
* Target position information must not be included inside the machine command.

Material normalization rules:

* Normalize the material into one of:
  "aluminum", "steel", "carbon_steel", "stainless_steel",
  "titanium", "iron", "MD_CFRP", "UD_CFRP".
* Recognize Korean names, English names, abbreviations, common misspellings, and speech-recognition variations.
* Ignore capitalization, spaces, hyphens, and underscores when comparing material names.
* Words such as "plate", "block", "sheet", "판", "봉", and "블록" describe the material shape and do not change the material type.
* Use the most specific material category when multiple names match.
* If the material is missing or cannot be identified confidently, return null.

Material mappings:

* Expressions containing "알루미늄", "알미늄", "aluminum", "aluminium", or clearly identified aluminum grades such as "A6061" and "A7075"
  -> "aluminum"

  -> "stainless_steel"

* "탄소강", "카본 스틸", "연강", "mild steel", "carbon steel", and carbon-steel grades such as "SS400" and "S45C"
  -> "carbon_steel"

* "강철", "강재", "일반강", "스틸", "steel", and generic machining expressions such as "철판"
  -> "steel"

* "티타늄", "타이타늄", "titanium", "Ti-6Al-4V", and "Ti64"
  -> "titanium"

* "순철", "순수 철", "pure iron", "elemental iron", and "Fe"
  -> "iron"

  -> "UD_CFRP"

* "MD CFRP", "MD-CFRP", "엠디 카본", "다방향 CFRP", "multidirectional CFRP", "woven CFRP", and "cross-ply CFRP"
  -> "MD_CFRP"

Material conflict rules:

* "stainless steel" must be classified as "stainless_steel", not "steel".
* "carbon steel" must be classified as "carbon_steel", not "steel".
* "carbon steel" must never be classified as CFRP.
* Generic "철", "철판", and "쇠" should normally be classified as "steel".
* Use "iron" only when pure or elemental iron is explicitly stated.
* Do not classify generic "CFRP", "카본", or "탄소섬유 복합재" as MD or UD unless the fiber direction is specified.


Process interpretation rules:

* Infer the machining process only when it is explicitly stated or unambiguously implied.
* Requests to make, drill, or bore a hole indicate "drilling".
* Requests to mill, cut, face, pocket, slot, or remove material using an endmill or ballmill indicate "milling".
* Requests to create an internal thread or perform tapping indicate "tapping".
* If a drill is explicitly specified as the cutting tool, classify the process as "drilling".
* If an endmill or ballmill is explicitly specified as the cutting tool, classify the process as "milling".
* The generic request "machine" or "process" alone is not sufficient to infer the process.
* Do not infer a process only from the material name, rpm, depth, or position.
* If the specified tool conflicts with the requested process, prioritize the explicitly requested process.
* If the process remains ambiguous, return null instead of guessing.

Tool interpretation rules:

* Normalize "엔드밀", "end mill", and "endmill" to "endmill".
* Normalize "드릴", "드릴 비트", "drill", and "drill bit" to "drill".
* Normalize "볼밀", "볼 엔드밀", "ball mill", and "ballmill" to "ballmill".
* Normalize "탭", "탭 공구", and "tap" to "tap".
* If the user requests drilling but does not explicitly specify the tool, infer "drill".
* If the user requests tapping but does not explicitly specify the tool, infer "tap".
* Do not infer "endmill" or "ballmill" unless the tool or operation clearly indicates it.

Diameter interpretation rules:

* When the user specifies a hole size, drill size, tool size, or diameter, return the value in "diameter".
* Expressions such as "5 mm hole", "5mm 구멍", "Ø5", "phi 5", "5파이", "drill 5 mm", and "5 mm drill" indicate a diameter of 5 mm.
* Treat the Korean expression "파이" as diameter.
* If "파이" has no explicit unit, interpret it as millimeters.
* Do not interpret a hole diameter as machining depth.
* If both diameter and depth are provided, return them separately.
* A value preceded by "깊이", "depth", or followed by "깊이로" must be stored in "depth", not "diameter".

Multiple command rules:

* If the user specifies both a target position and machining conditions, return both a "target" command and a "machine" command.
* Return commands in the order in which they should be executed.
* The "target" command should normally come before the "machine" command.
